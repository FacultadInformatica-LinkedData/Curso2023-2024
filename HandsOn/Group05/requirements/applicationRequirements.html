Application Requirements

1. Application Overview

Our application provides visitors of Paris valuable information for better urban experience. The information is related to air quality, green spaces, bicycle traffic, and bike-sharing stations.

2. User Requirements

  • Users should be able to access real-time air quality data in different areas of Paris.
  • Users should have access to information about nearby green spaces, parks, and recreational areas.
  • For cyclists, the application should offer real-time updates on bicycle traffic conditions and suggest optimal routes.
  • Users should be able to locate nearby bike-sharing stations and check bike availability.

3. Functional Requirements

  • Integration with air quality monitoring APIs to provide up-to-date air quality data.
  • Use of geospatial data to display green spaces on a map with details such as size, location, and available amenities.
  • Implementation of a bicycle traffic monitoring system using traffic sensors and real-time data analysis.
  • Integration with bike-sharing station APIs to display station locations and real-time bike availability.

4. Data Requirements

The application will rely on the following datasets:

  • Qualite-de-l-air-emissions-sectorielles-de-nox-pm2-5-pm10
  • Espaces_verts
  • Comptage-velo-compteurs
  • Velib-emplacement-des-stations
