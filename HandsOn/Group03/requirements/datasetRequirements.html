Dataset Requirements

Satisfacemos todos lo requisitos obligatorios de forma que:

R1. Los datos se encuentran en uno de los dominios de ciudad inteligente seleccionados: Utilizamos los datos de BiciMad(Madrid)
R2. Los datos están disponibles como un archivo CSV: Sí, lo están
R3. Los datos tienen una licencia abierta para que puedan ser publicados: Aviso legal de uso de datos públicos en Madrid
R4. Los datos se pueden vincular fácilmente con entidades genéricas del mundo real (por ejemplo, ubicaciones): Se nos proporcionan coordenadas por lo que se puede vincular fácilmente.

Requisitos opcionales:

R5. Existe documentación para los datos (opcional): Sí se cumple Documentación para los datos
R6. Los datos pueden provenir de múltiples fuentes de datos (opcional): No se cumple

Anotaciones importantes: En nuestro dataset no se especifica el nombre de la "smart city" pero sí se proporcionan las coordenadas por lo que hemos podido verificar que se trata de la ciudad de Madrid. Además, realizando esta comprobación nos hemos dado cuenta que las coordenadas de algunos datos son erróneas por el formato de las mismas, aun así creemos que podemos realizar el trabajo sin inconvenientes puesto que se puede arreglar manualmente.

Como se dijo antes sobre la licencia con la que se blindan los datos es la especificada en el requisito 3. Por otra parte, en cuanto a las entidades que se pueden identificar aquí para despues linkearlas con otros dataset serían el distrito, el barrio, la calle y la dirección.
