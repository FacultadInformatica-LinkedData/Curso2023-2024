Dataset analysis

Variable Type Range Description Problemas
Número String Unique Valor único que identifica cada uno de los puntos de bici Madrid Son identificadores problemáticos ya que no siguen un formato claro
Fecha de Alta Integer [41782- 44194] Dígito que identifica la fecha de alta del punto de bicis El formato de fecha de nuestra base de datos considera la fecha de origen del 1 de enero de 1900 como número 0. Por ejemplo, el número 43803 se refiere a la fecha específica del 26 de septiembre de 2020.
Distrito String Not Unique = Nª + Distrito Indica el número de distrito y el distrito de madrid
Barrio String Not Unique = Nª + Distrito Mas especifico indicando el barrio del distrito.
Calle String Not Unique Se es más concreto aun y se especifica la calle donde se ubique el punto de bicicletas En algunos casos el nombre no este completo
Nº Finca String Not Unique Es el número de la calle Como no se trata de un inmueble no tienen un número asignado como tal por tanto hay alguna fila sin valor o con valores extraños
Tipo de Reserva String Not unique Indica el modo de reserva para obtener la bicicleta Solo se ve un único valor posible así que no sabemos si prescindir de este atributo
Número de plazas Integer [3-30] Indica el número de bicicletas que alberga cada punto
Longitud Double [-3.73029/-3.6346859] Longitud de su ubicación Algunos valores tienen más puntos de sensibilidad que otros
Latitud Double [40.3889363/40.4810028] Latitud de su ubicación Algunos valores tienen más puntos de sensibilidad que otros
Dirección String Not Unique Alberga los atributos calle y número finca. Puede ser que el nombre de la calle no este completo y algunos no tienen un numero relacionado
Los atributos Gis_x y Gis_y hemos decidido no usarlos ya que no sabemos como poder interpretarlos .
